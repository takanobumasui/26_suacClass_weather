# 時計＋天気予報Webアプリ 入門メモ

HTML / CSS / JavaScript を使って、時計と天気予報を表示するWebアプリを作る方法をまとめました。
まずは「動くもの」を触ってみて、そのあと少しずつ仕組みを理解していけばOKです。

---

## 1. これって作れるの？

作れます。特別な知識やソフトは不要で、**HTML・CSS・JavaScriptの3つだけ**で完成します。

| 役割 | 担当 | たとえ |
|---|---|---|
| 構造(骨組み) | HTML | 家の柱・部屋の配置 |
| 見た目(装飾) | CSS | 壁紙・家具の配置 |
| 動き・データ処理 | JavaScript | 電気を通して家電を動かす |

柱がなければ壁紙は貼れないし、家がなければ家電も置けません。
なので理解する順番も **HTML → CSS → JavaScript** がおすすめです。

---

## 2. 天気データはどこから取ってくる？

**Open-Meteo**という、天気データを提供しているサービスを使います。

- 会員登録・APIキー取得が **不要**
- URLにアクセスするだけで天気データがもらえる
- 個人制作・学習用途にぴったり

試しにこのURLをブラウザで開いてみてください（浜松市付近の例）。

```
https://api.open-meteo.com/v1/forecast?latitude=34.98&longitude=138.38&current_weather=true
```

画面いっぱいに文字が表示されるはずです。これが **JSON** というデータ形式です。

```json
{
  "current_weather": {
    "temperature": 28.5,
    "windspeed": 12.3,
    "weathercode": 1
  }
}
```

`"名前": 値` のペアが並んでいるだけの、いわば「データのメモ」です。
人間が読んでもなんとなく意味がわかるように作られています。

---

## 3. JSONをどうやってWebページに表示する？

JavaScriptの `fetch` という命令を使います。`fetch`は英語で「取ってくる」という意味で、そのまま「データを取りに行く命令」です。

```javascript
const response = await fetch('さっきのURL');
const data = await response.json();

console.log(data.current_weather.temperature); // → 28.5
```

- `fetch(URL)` → そのURLにリクエストを送る（＝APIを「叩く」と言う）
- `await` → 返事が届くまでちゃんと待つ、という意味
- `response.json()` → 届いたデータをJavaScriptで使いやすい形に変換する
- `data.current_weather.temperature` → データの中身を`.`で辿って取り出す

エンジニアの間では「APIにリクエストを送ること」を **「APIを叩く」** と呼びます。キーボードを叩くという意味ではないので注意（最初は誰でも混乱するポイントです）。

## 4. HTML・CSS・JavaScriptの役割分担（サンプルコードより）

### HTML：表示する「箱」を用意するだけ

```html
<div id="clock">--:--:--</div>
<div id="temperature">--℃</div>
```

まだ中身は空っぽです。`id="clock"`のような名前をつけておくことで、あとでJavaScriptから「この箱を探して中身を書き換えて」と指示できるようになります。

### CSS：箱の見た目を整える

```css
#clock {
  font-size: 64px;
  color: #fff;
}
```

`#clock`のように`#`をつけて、HTML側の`id`と同じ名前を指定すると、その箱にだけスタイルが適用されます。

### JavaScript：箱の中身を埋める

```javascript
// 時計を1秒ごとに更新
function updateClock() {
  const now = new Date();
  document.getElementById('clock').textContent = now.toLocaleTimeString();
}
setInterval(updateClock, 1000);

// 天気を取得して表示
async function updateWeather() {
  const response = await fetch('Open-MeteoのURL');
  const data = await response.json();
  document.getElementById('temperature').textContent = data.current_weather.temperature + '℃';
}
updateWeather();
```

`document.getElementById('clock')`で、HTML側で作った箱を見つけてきて、`.textContent = ...`で中身を書き換えています。

---

## 5. 天気データはどのくらいの頻度で取得する？

天気予報は数分では変わらないので、頻繁に取得する必要はありません。

- **APIへの問い合わせ**：30分に1回くらいで十分
- 頻繁に叩きすぎると、無料APIの利用制限に引っかかることがある

```javascript
setInterval(updateWeather, 30 * 60 * 1000); // 30分ごと
```

---

## 6. 学習の進め方（おすすめ）

1. まず動くサンプルをLive Serverなどで実際に動かしてみる
2. HTMLファイルを見て、「どんな箱が用意されているか」を確認する
3. CSSファイルを見て、「箱の見た目がどう変わっているか」を確認する
4. JavaScriptファイルを見て、「箱の中身がどう埋まっていくか」を確認する
5. わからない単語（`fetch`、`API`、`JSON`など）は、その都度調べればOK。最初から全部理解しようとしなくて大丈夫です。

## 用語ミニ辞典

| 用語 | 意味 |
|---|---|
| API | あるサービスが「この手順でお願いすればデータをあげます」と決めた窓口 |
| JSON | `"名前": 値`の形でデータを表す形式。APIから返ってくるデータの多くはこの形 |
| fetch | JavaScriptでデータを取りに行くための命令 |
| 叩く（APIを） | 「APIにリクエストを送る」という業界スラング |
| async / await | 時間のかかる処理（通信など）を、上から順番に読める形で書くための書き方 |
| id / class | HTMLの箱につける名前。`id`はページに1つだけ、`class`は複数の箱につけられる |
